================================================================================
                        API ENDPOINTS
================================================================================

USER MANAGEMENT (PostgreSQL + Neo4j)
  POST   /users                     Create user (auto-sync to Neo4j)
  GET    /users                     List all users
  PUT    /users/:id                 Update user
  DELETE /users/:id                 Delete user (cascade in Neo4j)

POST MANAGEMENT (MongoDB + Enrichment)
  POST   /posts                     Create post
  GET    /posts                     List posts (enriched with user data)
  PUT    /posts/:id                 Update post
  DELETE /posts/:id                 Delete post

GRAPH RELATIONSHIPS (Neo4j)
  POST   /follow                    Create follow relationship
  GET    /relationships             Get relationships with names

ANALYTICS (TimescaleDB)
  GET    /analytics                 Get event statistics

CACHE MONITORING (Redis)
  GET    /cache-status              Get Redis health metrics

PREREQUISITES:
  ‚Ä¢ Docker Desktop (v20.10+)
  ‚Ä¢ Docker Compose (v2.0+)
  ‚Ä¢ 8GB RAM minimum
  ‚Ä¢ 10GB free disk space

START THE SYSTEM:
  cd Advanced-Database-Project
  docker-compose up -d

VERIFY SERVICES:
  docker-compose ps

ACCESS APPLICATIONS:
  Frontend:      http://localhost:3000
  API:           http://localhost:4000
  Neo4j Browser: http://localhost:7474 (neo4j / neo4jpass)

EXAMPLE REQUESTS:
  # Create user
  curl -X POST http://localhost:4000/users \
    -H "Content-Type: application/json" \
    -d '{"email":"user@example.com","name":"John","password":"pass"}'

  # Create post
  curl -X POST http://localhost:4000/posts \
    -H "Content-Type: application/json" \
    -d '{"author_id":"<uuid>","title":"My Post","body":"Content"}'

  # Create relationship
  curl -X POST http://localhost:4000/follow \
    -H "Content-Type: application/json" \
    -d '{"followerId":"<uuid1>","followeeId":"<uuid2>"}'

  # Get analytics
  curl http://localhost:4000/analytics

  # Get graph relationships
  curl http://localhost:4000/relationships
